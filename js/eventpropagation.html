Event propagation defines the order in which events are received by the target element and its ancestors in the DOM
        hierarchy. There are three phases in event propagation:
        
        1. => Capturing Phase (Event Capturing):
        
        The event starts from the window and travels down to the target element.
        
        It's less commonly used but can be handled by setting the third argument of addEventListener to true.
        
        2. =>Target Phase:
        
        The event reaches the target element.
        
        3. => Bubbling Phase (Event Bubbling):
        
        The event bubbles up from the target element to its ancestors in the DOM tree.
        
        This is the default phase when events are handled.
